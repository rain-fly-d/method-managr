# 方法管理 —— 用文件名打开任意文件/文件夹

> 一个通过重命名 exe 来触发不同功能的趣味工具，灵感来源于游戏《windows is white》的“改名破关”机制。

## 📖 项目简介

当你需要隐藏一些文件的入口时，可能会用到它：  
- 在 `学习资料.exe` 下，直接打开学习资料文件夹；  
- 在 `2077.exe` 下，直接打开《赛博朋克2077》游戏（即使游戏本身的 `.exe` 已被改成乱码名称，避免被系统搜索到）。  

**方法管理** 提供了一个极简方案：你只需将同一个 exe 文件重命名为预设的名称，双击后就会自动打开对应的目标路径。它既可以用作图形化的映射管理工具，也可以作为一个“可重命名的万能启动器”。

## ✨ 核心特性

### 两种运行模式
1. **配置模式**：文件名 = `方法管理.exe` → 打开图形界面，管理文件名与目标路径的映射。  
2. **执行模式**：文件名 = 配置中的任意名称 → 直接打开对应文件/文件夹，然后自动退出。

### 功能亮点
- **动态热切换**：在配置模式运行期间，直接重命名该 exe 文件，程序会自动检测并打开目标路径（窗口不关闭），实现“无重启即用”。  
- **持久化存储**：映射数据保存在同目录下的 `config.json` 中，便于备份和迁移。

## 🚀 快速开始

### 1. 初次使用（配置模式）
- 双击 `方法管理.exe` 打开配置窗口。  
- 在底部输入框中：  
  - **文件名**：填写希望用于启动的文件名（例如 `我的文档.exe`，程序会自动补充 `.exe` 后缀）。  
  - **目标路径**：填写要打开的文件夹或文件的完整路径（例如 `C:\Users\你的用户名\Documents`）。  
- 点击 **添加** 保存映射。  
- 重复以上步骤可添加多个映射。所有映射会显示在列表中，选中后可在输入框编辑。

### 2. 使用映射启动
- 将 `方法管理.exe` 重命名为你配置中设置的文件名（如 `我的文档.exe`）。  
- 双击该文件，目标文件夹会立即打开，程序随后自动关闭。

### 3. 动态切换演示
- 先打开配置窗口（`方法管理.exe` 保持运行）。  
- 在资源管理器中，将 `方法管理.exe` 重命名为另一个已配置的文件名（如 `代码.exe`）。  
- 程序检测到文件名变化，自动打开对应目录，但窗口不会关闭，继续等待下一次重命名。

## ⚙️ 配置文件说明

- `config.json` 与主程序位于同一目录，格式如下：  
  ```json
  {
      "我的文档.exe": "C:\\Users\\你的用户名\\Documents",
      "代码.exe": "D:\\Projects"
  }
  ```
- 你可以手动编辑该文件，但推荐使用图形界面以避免格式错误。

## ❓ 常见问题

**Q: 为什么重命名后双击没反应？**  
A: 请确保：  
- 文件名完全匹配配置中的名称（包括大小写，但程序会忽略大小写差异）。  
- 目标路径真实存在。  
- 程序文件所在目录有写入权限（例如放在 `Program Files` 中可能导致配置无法保存）。

**Q: 如何在执行模式下不自动退出？**  
A: 执行模式默认打开目标后退出，如需保持运行请使用配置模式并动态重命名。

**Q: 可以打开程序而不是文件夹吗？**  
A: 可以。目标路径填写程序的完整路径（如 `C:\Windows\System32\notepad.exe`）即可启动该程序。

**Q: 文件名一定要带 `.exe` 后缀吗？**  
A: 是的。程序会自动为输入的文件名补全 `.exe`，但最终配置中的名称必须包含 `.exe`（例如 `计算器.exe`）。

## 🛠️ 技术实现

本项目基于 **Python + Tkinter** 开发，通过 **PyInstaller** 打包为独立 exe。  
核心机制：定时扫描当前目录下所有 `.exe` 文件，取最新修改的文件名，匹配配置后执行打开操作。  
这种方式绕过了 Windows 对运行中进程文件名的缓存限制，实现了“运行时重命名即响应”。
